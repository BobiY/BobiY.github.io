什么是函数式编程

简单来说，函数式编程是一种编程范式，也就是如何编程的一种方法。它属于结构化编程的一种，就是将运算过程通过一系列的函数嵌套完成。这样说起来可能太抽象，看下面的一个例子：

{% highlight javascript %} // 此处有这样一个数学运算 // ( 1+2 ) * 3 -4 // 用函数表示他的运算过程就是 subtract( multiply( add( 1,2 ),3 ), 4) {% endhighlight %}

上面的函数表示运算过程，并且能计算出最终的结果，这就是函数编程；这只是简单的演示，在真正的了解函数式编程之前，先来了解一下纯函数和柯里化这两个概念，它将是函数式编程的常用和重要的方法

纯函数

纯函数就是在相同输入的情况下，得到的输出结果也是相同的。它的输出结果只取决于你传入的参数而不受外界环境的影响，看例子：

{% highlight javascript %} // 定义一个函数 function aa(state){ var a = state; return a + 1; } aa( 1 ) // 2; var c = 3; function dd(state){ return c + state; } dd( 1 ) // 4; {% endhighlight %}

上面例子中，函数 aa 就属于一个纯函数，因为他的输出只依赖于它的参数 state 而与外界的环境无关，这样在输入相同时，我们就可以拿到相同的输出值，这样，整个应用的状态都是可预测的； 而函数 dd 的输出依赖于全局变量 c ，这样子，在不知道 c 的值得情况下，函数的输出值就不仅仅是取决于他的参数，因为 c 的值是未知的而且是可变的。

柯里化

柯里化是函数有多个参数时，将参数分开传递，利用闭包的原理时内部函数可以使用所有参数的处理方法

{% highlight javascript %} // 简单例子 function a(a){ return function b(b){ return function c(c){ return a + b + c; } } } {% endhighlight %}

函数式编程的特点

1.函数是“一等公民”

这里说的意思就是，函数和其他数据类型一样，可以作为参数传递，可以赋值给其他变量，这在 js 中是很常见的用法

只用“表达式”，不用“语句”

只用表达式的意思就是每个函数必须有返回值，而你仔细想想自己接触过的表达式，基本都是会给你返回一个结果；而语句仅仅是执行某种操作，然后你不能通过这个执行的函数拿到返回值，就像 if 语句一样，你只能在其他地方看到执行结果，但是 if 语句本身不会给你返回值。因为函数式编程是将运算过程用函数表达出来，所以每个参数计算的函数，我们都需要他们的返回值去做下一步的计算 或者拿到最终的结果。

没有副作用

所谓"副作用"（side effect），指的是函数内部与外部互动（最典型的情况，就是修改全局变量的值），产生运算以外的其他结果。 函数式编程强调没有"副作用"，意味着函数要保持独立，所有功能就是返回一个新的值，没有其他行为，尤其是不得修改外部变量的值。

不修改状态

上一点已经提到，函数式编程只是返回新的值，不修改系统变量。因此，不修改变量，也是它的一个重要特点。

5.引用透明

引用透明（Referential transparency），指的是函数的运行不依赖于外部变量或"状态"，只依赖于输入的参数，任何时候只要参数相同，引用函数所得到的返回值总是相同的。 有了前面的第三点和第四点，这点是很显然的。其他类型的语言，函数的返回值往往与系统状态有关，不同的状态之下，返回值是不一样的。这就叫"引用不透明"，很不利于观察和理解程序的行为。

为什么要使用函数式编程

1.代码简洁，开发速度快

函数式编程大量使用函数，减少了代码的重复，因此程序比较短，开发速度较快。

2. 接近自然语言，易于理解

函数式编程因为编程的主导是函数，而且计算过程都是由函数组成，所以看起来会更加的贴近我们平时的数学计算，拿一个简单的计算 ( 1+1 )*2-3来说：

{% highlight javascript %} subtract(multiply(add(1,1), 2), 3) {% endhighlight %}
